## Boolean format (L061)

Use `true`/`false` for booleans, not `yes`/`no`/`True`/`False`.

YAML parsers convert unquoted `yes`/`no` to native booleans before the AST is
built. This rule catches quoted truthy strings (e.g. `"yes"`, `"True"`) that
survive YAML parsing and reach OPA as string values. The ARI engine normalizes
quoted truthy strings to native booleans, so these examples cannot be tested
through the integration harness. OPA unit tests in `L061_test.rego` validate
the Rego logic directly.

### Example: violation

```yaml
- name: Restart httpd
  ansible.builtin.service:
    name: httpd
    state: restarted
  become: "yes"
```

### Example: pass

```yaml
- name: Restart httpd
  ansible.builtin.service:
    name: httpd
    state: restarted
  become: true
```
