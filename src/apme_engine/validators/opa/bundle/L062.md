## YAML module args (L062)

Use YAML-style module arguments instead of `key=value` one-liners.

Detects `_raw_params` containing `=` in the OPA AST. Excludes command/shell/raw
modules where free-form syntax is expected. The ARI engine expands `key=value`
free-form syntax into separate module options rather than preserving
`_raw_params`, so these examples cannot be tested through the integration
harness. OPA unit tests in `L062_test.rego` validate the Rego logic directly.

### Example: violation

```yaml
- name: Install nginx
  ansible.builtin.apt: name=nginx state=present
```

### Example: pass

```yaml
- name: Install nginx
  ansible.builtin.apt:
    name: nginx
    state: present
```
