## Argspec validation — docstring (L058)

Parses each module's `DOCUMENTATION` string (via `ansible.utils.plugin_docs.get_docstring`) to extract the argument specification, then validates user-supplied parameters against it. Safe (no module code execution), fast, but may drift from the actual `argument_spec` defined in code.

Checks: unsupported parameters, missing required parameters, invalid choices.

### Example: violation

```yaml
- name: Example play
  hosts: localhost
  connection: local
  tasks:
    - name: Bad args
      ansible.builtin.file:
        path: /tmp/foo
        invalid_param: true
```

### Example: pass

```yaml
- name: Example play
  hosts: localhost
  connection: local
  tasks:
    - name: Good args
      ansible.builtin.file:
        path: /tmp/foo
        state: touch
```
