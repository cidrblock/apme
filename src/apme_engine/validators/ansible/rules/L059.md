## Argspec validation — mock/patch (L059)

Loads the module's Python file, patches `AnsibleModule.__init__` to capture the real `argument_spec`, then validates user-supplied parameters against it. More accurate than L058 — catches `mutually_exclusive`, `required_together`, `required_if` — but executes module import code.

Checks: unsupported parameters, missing required parameters, invalid choices, mutually exclusive groups, required-together groups.

### Example: violation

```yaml
- name: Example play
  hosts: localhost
  connection: local
  tasks:
    - name: Bad args
      ansible.builtin.file:
        path: /tmp/foo
        invalid_param: true
```

### Example: pass

```yaml
- name: Example play
  hosts: localhost
  connection: local
  tasks:
    - name: Good args
      ansible.builtin.file:
        path: /tmp/foo
        state: touch
```
