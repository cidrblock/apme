## Syntax check (L057)

Runs `ansible-playbook --syntax-check` on each playbook found under the project root. Reports any syntax errors detected by Ansible itself (e.g. invalid YAML, unknown modules at parse time, missing imports).

### Example: violation

```yaml
- name: Bad playbook
  hosts: localhost
  connection: local
  tasks:
    - name: Broken task
      ansible.builtin.debug:
        msg: {{ unquoted_jinja }}
```

### Example: pass

```yaml
- name: Good playbook
  hosts: localhost
  connection: local
  tasks:
    - name: Working task
      ansible.builtin.debug:
        msg: "Hello world"
```
